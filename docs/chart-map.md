# 차트·컨트롤 계약

화면은 `dashboard.json`·`history.json`·`strategy-comparison.json`에 발행된 파생 결과와 입력 계약만 사용한다. 기본 80/20 결과는 서버 검증값을 선택하고, 다른 사용자 청산선은 공개된 신호·거래 가능 여부·조정가격으로 거래 경로만 브라우저에서 계산한다. 브라우저는 회귀·잔차·백분위·상태·사건 표본을 다시 적합하거나 임계값을 최적화하지 않는다. 지원하지 않는 조합은 비활성화하고 `unavailable`을 표시하며 임의의 시장 수치로 대체하지 않는다.

## 연구 트랙과 적용 범위

| 트랙 | 역할 | 입력·추정 | 기본 사용처 | 화면 표현 |
| --- | --- | --- | --- | --- |
| `robust` | 실전 신호 | 시장 규모 보정 수급 · 과거 데이터만 사용한 강건 회귀 | v3 현재값, 잔차, 사건, 기본 전략 | `실전 신호` 배지와 원형 keyline |
| `scaled` | 연구 기준선 | 시장 규모 보정 수급 · 과거 252일 OLS | 강건 회귀와 비교하는 기준선 | `OLS 기준선` 배지와 청록 keyline |
| `raw` | PDF 원문 근사 | 개인 순매수대금 절대값 · 과거 회귀 | 원문 주석 날짜와 방향 대조 | `PDF 원문 근사` 배지와 마름모 keyline |

`raw`는 원문 회귀 기간·이상점 제외·경계식이 공개되지 않았으므로 “원문 충실” 또는 “완전 재현”이라고 부르지 않는다. v1 payload에서는 `robust`가 없으므로 `scaled`로 안전하게 내려간다. 사건 연구는 `eventsByModel`이 있을 때 연구 트랙 선택을 따르고, 없으면 공개된 기본 사건 결과에 고정한다. 결론 배지는 사건 자산·연구 트랙·표본과 전략 프록시·규칙을 따로 표시한다. 전략은 별도의 신호·진입 규칙 선택을 사용한다.

## 컨트롤과 상태 보존

- 연구 트랙: 실전 강건 회귀 / 규모보정 OLS 기준선 / 절대수급 PDF 원문 근사
- 사건: KOSPI / 226490 / 069500, 전체 / 20일 비중첩
- 전략: 포지션 정책(나란히 / 롱·현금 / 롱·숏·현금), 프록시, 실전 강건 회귀 / OLS 기준선 / raw 원문 근사 / 이격도 강건성, 편도 0 / 5 / 10 / 20bp, 전체 / 공통기간
- 사용자 청산 기준: 롱 정수 50~94, 숏은 `100 - 롱` 대칭값. 기본 80/20은 서버 검증, 나머지는 브라우저 사용자 시나리오다. 이격도 진입 변형은 대칭 숏 규칙이 없어 롱/숏 정책에서 사용할 수 없다.
- KOSPI 가격 표시 기간: 기준일에서 달력 기준으로 계산한 1개월 / 3개월 / 6개월 / YTD / 1년 / 3년 / 전체, 또는 사용자가 지정한 시작일·종료일. 요청 경계가 휴장일이면 그 안의 실제 첫·마지막 거래일을 사용하고 화면에 표시 범위와 거래일 수를 공개한다. 520px 이하에서 처음 방문하면 1년으로 시작한다.
- 사용자 지정 기간은 시작·종료를 포함하며 역전, 공개 이력 밖, 거래일 0건을 거부한다. 8거래일 미만이면 표는 유지하고 추세 차트만 부족 안내를 표시한다. KOSPI 기간 선택은 잔차·회귀·사건·전략을 다시 계산하지 않는다.
- 잔차 표시 기간은 최신 756거래일(약 3년)로 고정한다.
- 모든 선택은 `fearngreed-controls-v2` localStorage와 URL query에 저장한다. “기본값 복원”과 “현재 화면 링크 복사”를 제공한다.
- v3 `history.seriesColumns` + `seriesRows`의 모델별 거래 가능 열과 `strategyScenario` 계약을 사용한다.
- 표와 차트는 문서 전체가 아니라 자체 영역만 가로 스크롤하며, 모바일 시계열은 최신 구간으로 자동 정렬한다.

## 시각·접근성 원칙

Quant Research 공통 파랑을 기본 인터랙션과 롱/현금 경로에, 청록을 기준선·매수보유 계열에, 주황을 원문 근사·탐욕·롱/숏 경로에 쓴다. 색만으로 상태와 정책을 구분하지 않고 원·마름모·사각형, 실선·점선, 숏 노출 사선 패턴, 직접 텍스트와 번호를 함께 사용한다. 합성 숏의 제외 가정은 전략 영역에서 항상 보이게 두며 접힌 상세 설명에만 숨기지 않는다. 모든 조작 버튼은 44px 내외의 터치 영역과 `focus-visible`을 유지하며 360px에서 입력·버튼·노출 카드가 한 열로 접히고 표·차트만 자체 가로 스크롤한다. `prefers-reduced-motion`에서는 부드러운 스크롤·전환을 제거한다. 운영 상태 배지와 연구 신호 배지는 별도 DOM·문구·스타일로 표시한다.
