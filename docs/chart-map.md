# 차트·컨트롤 계약

화면은 `dashboard.json`·`history.json`·`strategy-comparison.json`에 발행된 파생 결과와 입력 계약만 사용한다. 페이지 권장 시작값은 절대수급·196일·R² 0.40·극단 2%·최대 20일·청산 80/20·YTD·전체 사건이며 최소 157개 관측을 사용한다. 서버 정식 산출의 252일·R² 0.20·극단 5% 기준은 그대로 유지한다. 사용자가 값을 바꾸면 브라우저가 공개 일별 입력으로 과거 전용 회귀·잔차·백분위·상태·사건 표본·전략 경로를 함께 다시 계산하되 임계값을 자동 최적화하지 않는다. 지원하지 않는 조합은 비활성화하고 `unavailable`을 표시하며 임의의 시장 수치로 대체하지 않는다.

## 연구 트랙과 적용 범위

| 트랙 | 역할 | 입력·추정 | 기본 사용처 | 화면 표현 |
| --- | --- | --- | --- | --- |
| `robust` | 실전 신호 | 시장 규모 보정 수급 · 사용자 학습창의 과거 전용 강건 회귀 | v5 선택 기준일, 잔차, 사건, 기본 전략 | `실전 신호` 배지와 원형 keyline |
| `scaled` | 연구 기준선 | 시장 규모 보정 수급 · 사용자 학습창의 과거 전용 OLS | 강건 회귀와 비교하는 기준선 | `OLS 기준선` 배지와 청록 keyline |
| `raw` | 규모 미보정 비교 | 개인 순매수대금 절대값 · 과거 전용 OLS | 명목 원화 수급의 독립 비교 | `절대수급 OLS` 배지와 마름모 keyline |

`raw`는 시장 거래대금으로 나누지 않은 명목 원화 수급을 묻는 독립 트랙이다. 사건 연구와 전략은 상단의 동일한 연구 트랙·학습창·품질 기준·극단 꼬리·평가 기간을 따르며, 결론 배지는 사건 자산·표본과 전략 프록시·규칙을 구분해 표시한다.

## 컨트롤과 상태 보존

- 연구 트랙: 실전 강건 회귀 / 규모보정 OLS 기준선 / 절대수급 OLS
- 사건: KOSPI / 226490 / 069500, 전체 / 20일 비중첩
- 전략: 포지션 정책(나란히 / 롱·현금 / 롱·인버스·현금), 실제 ETF 페어(1X 069500·114800 / 2X 122630·252670), 실전 강건 회귀 / OLS 기준선 / 절대수급 OLS / 이격도 강건성, 편도 0 / 5 / 10 / 20bp, 전체 / 공통기간
- 통합 사용자 시나리오: 연구 트랙, 회귀창 60~756거래일, 최소 R² 0~0.8, 극단 꼬리 1~20%, 최대 보유 1~60거래일과 청산 기준을 함께 적용한다. 롱 청산은 정수 50~94, 인버스는 `100 - 롱` 대칭값이다. 페이지 권장 시작값은 절대수급·196일·0.40·2%·20일·80/20이고, 252일·0.20·5%·20일·80/20은 서버 정식 산출 기준이다. 이격도 진입 변형은 대칭 인버스 규칙이 없어 롱/인버스 정책에서 사용할 수 없다.
- KOSPI 가격 표시 기간: 기준일에서 달력 기준으로 계산한 1개월 / 3개월 / 6개월 / YTD / 1년 / 3년 / 전체, 또는 사용자가 지정한 시작일·종료일. 요청 경계가 휴장일이면 그 안의 실제 첫·마지막 거래일을 사용하고 화면에 표시 범위와 거래일 수를 공개한다. 화면 크기와 관계없이 처음 방문하면 최신 기준일까지 포함하는 YTD로 시작한다.
- 사용자 지정 기간은 시작·종료를 포함하며 역전, 공개 이력 밖, 거래일 0건을 거부한다. 8거래일 미만이면 표는 유지하고 추세 차트만 부족 안내를 표시한다. 엔진은 선택 시작일 이전 전체 공개 이력부터 과거 행만으로 신호와 포지션을 계산하고, 선택 기간은 평가 구간으로만 적용한다.
- 잔차·사건·전략 화면은 선택한 사용자 시나리오와 평가 구간을 반영하되, 서버 발행 정식 스냅샷과 기본 사건 연구를 덮어쓰지 않는다.
- 모든 선택은 `fearngreed-controls-v7` localStorage와 URL query에 저장한다. 이전 저장값은 호환해 읽고, 이전 기본 시나리오만 새 권장 시작값으로 옮기며 사용자가 바꾼 값은 보존한다. “기본값 복원”과 “현재 화면 링크 복사”를 제공한다.
- v5 `history.seriesColumns` + `seriesRows`의 원정밀도 입력, 네 실제 ETF의 조정 OHLC와 `strategyScenario` 계약을 사용한다.
- 표와 차트는 문서 전체가 아니라 자체 영역만 가로 스크롤하며, 모바일 시계열은 최신 구간으로 자동 정렬한다.

## 시각·접근성 원칙

Quant Research 공통 파랑을 기본 인터랙션과 롱/현금 경로에, 청록을 기준선·매수보유 계열에, 주황을 절대수급·탐욕·롱/인버스 경로에 쓴다. 색만으로 상태와 정책을 구분하지 않고 원·마름모·사각형, 실선·점선, 인버스 노출 사선 패턴, 직접 텍스트와 번호를 함께 사용한다. 실제 인버스 ETF와 2X 일간 배율의 경로의존성은 전략 영역에서 항상 보이게 두며 접힌 상세 설명에만 숨기지 않는다. 모든 조작 버튼은 44px 내외의 터치 영역과 `focus-visible`을 유지하며 360px에서 입력·버튼·노출·현재 거래 카드가 한 열로 접히고 표·차트만 자체 가로 스크롤한다. `prefers-reduced-motion`에서는 부드러운 스크롤·전환을 제거한다. 운영 상태 배지와 연구 신호 배지는 별도 DOM·문구·스타일로 표시한다.
