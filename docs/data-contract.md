# 데이터 및 공개 계약

KRX Open API의 KOSPI·ETF·유가증권 가격·거래대금은 1차 소스, 인증된 pykrx의 KOSPI 참여자별 순매수는 보완 소스, yfinance 조정가격은 연구용 2차 소스다. Open API 서비스 활용권한이 없거나 인증에 실패하면 인증된 pykrx KRX 가격 경로로 fail-closed 전환하되 공개 상태를 `degraded`로 두고 이유를 남긴다. 이를 Open API와 동등한 정상 상태로 표시하지 않는다. KOSPI와 수급의 최신일이 다르면 미래 행을 오래된 행에 붙이지 않고 최신 공통 거래일까지 절단하며 시차 사유를 공개한다. ETF 공식 전체 이력은 거래 세션과 다중 시점 진단에 사용하고, 최신 `dataAsOf` 종가 차이가 0.5% 이내일 때만 관련 백테스트를 공개한다. yfinance 조정가격에 공식 KRX 거래 세션이 빠진 경우 앞뒤 공통일의 `조정종가/원종가` 계수가 0.5% 이내로 일치할 때만 KRX OHLC를 같은 조정 스케일로 보정한다. 한 건이라도 보정할 수 없으면 해당 ETF 사건 연구와 백테스트 전체를 `unavailable`로 닫고, 보정·미해결 건수와 합성 출처를 공개한다. 원종가와 조정가격의 장기 차이는 배당·분할 효과가 있으므로 진단 정보이며 최신 게이트를 대체하지 않는다. 전체 백테스트는 배당·분할 반영 조정가격을 사용한다. ETF/ETN/ELW 수급은 제외한다. 미국 종가는 해당 KRX 거래일 전에 이용 가능했던 마지막 세션으로만 정렬한다.

`KRX_API_KEY`, `KRX_ID`, `KRX_PW`는 GitHub Actions Secrets 또는 일회성 환경변수로만 주입한다. 인증 구간의 stdout/stderr는 캡처하며 요청 헤더, 쿠키, 원응답과 예외 객체를 공개하지 않는다. `references/private/`, `.env*`, 세션 파일과 원시 인증 응답은 Git 및 Pages 산출물에서 제외한다.

`summary.json`은 `schemaVersion=1`, `contract=quant-research-summary`, `projectId=fearngreed`, `methodologyVersion=fear-flow-v5`을 고정한다. 엄격한 JSON Schema가 상태·범위·엔티티·robust/scaled/raw 최신 모형·기본 1X 실제 ETF 포지션·대기 주문·출처·자동화·payload의 필드, 범위, enum과 추가 속성을 검증한다. 최신성은 `generatedAt`이 아니라 `dataAsOf`로 판단한다. `status.expectedDataAsOf`는 KRX Open API가 확인한 최신 완료 세션이고, `freshnessBasis=official_krx_latest_completed_session`일 때 `sourceFreshnessPassed`는 KOSPI와 개인수급이 그 세션까지 모두 도착했을 때만 참이다. 공식일 미확정 상태는 `source_alignment_only`로 구분한다. 브라우저는 이 서버 판정을 우선하면서도 `automation.lastSuccessAt` 감시를 유지해 예약 실행 자체가 멈춘 경우를 `stale`로 표시한다. 화면은 `summary.json`·`dashboard.json`·`history.json`·`strategy-comparison.json`의 방법론 버전과 기준일이 모두 일치하지 않으면 `unavailable`로 닫히며 임의 시장 수치로 대체하지 않는다. 이 네 파일의 수치와 상태는 서버가 252일·최소 R² 0.20·극단 5%·최대 보유 20일·청산 80/20 기본값으로 발행한 정식 결과이며, 사용자가 브라우저에서 만든 시나리오와 구분한다.

`dashboard.json`은 모델별 252개 학습 관측과 현재 관측 1개의 역할, 무조건부 벤치마크·초과수익을 포함한 정식 사건 요약, ETF·비용·변형·기간별 기본 80 롱/현금 백테스트, PDF 사례 격리 패널, 수급 채널 확장 상태, 공식 종가 교차검증과 반도체 상대 진단을 둔다. `scatterMetaByModel.<model>.stateBoundaries`에는 최신 정식 회귀로 다시 계산한 학습 잔차의 경험적 CDF 전이 순서통계량을 공개한다. `residualOffsets`의 `extremeFearUpper`, `fearUpper`, `greedLower`, `extremeGreedLower`를 현재 회귀선에 더한 평행선만 정식 산점도 상태 영역으로 쓸 수 있으며, 브라우저는 반올림된 산점도 점으로 경계를 다시 추정하지 않는다. 사용자 시나리오 경계는 `history.json`의 원정밀도 입력에서 별도로 적합한다. 최신 정식 모형 상태가 `unavailable`이면 정식 경계도 `null`로 닫는다. 각 정식 사건 섹션의 `meanExcessReturnCi95BenchmarkTreatment`은 `fixed_external_mean`, `paired_event_returns`, `unavailable` 중 하나다. `fixed_external_mean`은 무조건부 벤치마크 평균을 고정한 조건부 bootstrap 구간으로 벤치마크 평균의 추정 불확실성을 포함하지 않는다는 뜻이다. 같은 정책을 상태·기간별 행마다 반복하지 않고 섹션 수준에 한 번 기록한다. 브라우저가 사용자 시나리오 신호로 다시 만든 사건 표는 고정 seed iid bootstrap을 사용하며 서버의 이동블록 사건 연구와 구분한다.

`history.json`은 `columnar-v1` 열·행 인코딩으로 일별 공개 입력과 정식 파생 시계열, 세 모델의 최신 스냅샷, 모델별 정식 거래 가능 여부와 통합 사용자 시나리오 계약을 공개한다. `p069500Open/Close`, `p114800Open/Close`, `p122630Open/Close`, `p252670Open/Close`는 실제 1X·2X 페어의 조정 체결·평가가격이며 `p226490Open/Close`는 기존 KOSPI 프록시 진단 호환용이다. `numericPrecisionDigits=8`은 공개 시계열의 부동소수점만 소수 8자리로 직렬화한다는 뜻이며 날짜·상태·불리언·출처 해시는 변경하지 않는다. `strategyScenario.browserMayRefitRegression=true`는 브라우저가 공개 수익률·개인 순매수로 사용자 시나리오의 회귀를 다시 적합할 수 있음을 뜻하며, 서버 발행 결과를 수정한다는 뜻이 아니다. `configurableInputs`는 회귀창 60~756(기본 252), 최소 R² 0~0.8(기본 0.20, 0.05 간격), 극단 꼬리 1~20(기본 5), 최대 보유 1~60(기본 20)의 범위·간격을 선언한다. 최소 학습 관측치는 `min(lookback,max(40,min(200,ceil(lookback*0.8))))`로 고정하며 기본 252일은 서버와 같은 최소 200개 관측을 요구한다. `pastOnly=true`는 각 거래일보다 앞선 행만 학습에 쓴다는 뜻이고, `evaluationRangeSeparate=true`는 사용자 평가 시작·종료일이 학습 이력을 자르거나 미래 행을 허용하지 않는다는 뜻이다. 페어의 한쪽이라도 상장 전이거나 가격이 없으면 그 행을 현금으로 간주하지 않고 `unavailable`로 둔다. 독립 소비자가 외국인·기관의 상태 열을 거래 신호로 오해하지 않도록 `flowChannelRoles`에 개인=`primary`, 외국인·기관=`diagnostic_only`, `eligibleForTrading=false`, 활성화 규칙과 실제 열 이름을 함께 둔다.

`strategy-comparison.json`은 `contract=fearngreed-strategy-comparison`으로 robust 기본 80/20 실제 롱/인버스/현금 결과, 1X·2X 페어 정의, 50 대 80 고정 민감도와 동적 입력 범위 50~94를 공개한다. `policyDefinitions.longInverseCash`는 실제 상장 ETF를 양의 수량으로 보유하는 정식 정책이고, `longShortCash`와 `legacyProxyContract`는 이전 소비자 호환용 비정식 진단이다. `dynamicExitControl`은 `inverseExitFormula=100-longExitPercentile`, 회귀 재적합 위치 `browser_on_server_published_history_and_adjusted_prices`, `regressionRefit=true`를 선언하고 `history.strategyScenario`과 동일한 가변 입력 범위·최소 관측 공식·`pastOnly`·`evaluationRangeSeparate` 값을 가져야 한다. `scenarioAuthority=browser_user_scenario_not_canonical_server_output`은 브라우저 결과가 정식 서버 산출물이나 최적화 결과가 아니라는 기계 판독 가능한 경계다. 실제 인버스 ETF는 음의 수량이나 합성 숏으로 계산하지 않으며 114800 또는 252670의 조정가격을 양의 수량으로 보유한다. 2X 결과는 기간수익률을 2배로 곱하지 않고 122630·252670 자체 가격 경로를 사용한다. 화면과 독립 소비자는 이를 기초지수의 정확한 기간 ±1배·±2배 또는 공매도 성과로 바꾸어 부르면 안 된다. `dashboard.json`과 `strategy-comparison.json`은 각각 500KB, `history.json`은 2MB 미만을 목표로 하며 검증기가 최소 5% 여유까지 강제한다.

갱신은 최신 5거래일을 수정 가능 구간으로 삼아 캐시를 강제 재검증하고, 수정 불가 구간의 모든 공개 열이 재계산 결과와 일치하는지 확인한다. 공개 입력을 다시 읽는 과정에서 robust z-score에 증폭되는 부동소수점 잡음만 절대오차 `5e-8`까지 허용하며 문자열·상태·포지션은 정확히 같아야 한다. Yahoo 조정가격은 경계 이전의 최대 6개 공통 앵커도 비교해 0.5%를 넘는 스케일 변경을 탐지한다. 이전 공개 이력에 검증된 KRX 보정행이 있으면 그 수와 출처를 다음 증분 실행에도 계승해, 당일 신규 보정이 0건이라는 이유로 순수 Yahoo 이력처럼 표시하지 않는다. 어느 검사든 실패하면 `requires_backfill`로 중단하며, 명시적 `backfill_start_date` 없이는 과거 원자료·신호·포지션을 일부만 고쳐 서로 다른 세대의 결과를 섞지 않는다. 검증된 결과는 임시파일을 완전히 쓰고 `fsync`한 뒤 원자적으로 교체한다. 실패하면 기존 신호·사건·백테스트를 유지하고 `summary.status`와 `automation-status.json`에 승인된 짧은 사유만 원자적으로 기록한다. 실패 실행도 이 두 파일만 검증·커밋해 공개 상태를 알린 뒤 Actions 자체는 실패로 남긴다. Pages 산출물은 `index.html`, `assets/`, `data/`, `docs/`, `schemas/`만 허용하며 `references/`는 복사하지 않는다.
