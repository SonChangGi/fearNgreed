# 데이터 및 공개 계약

KRX Open API의 KOSPI·ETF·유가증권 가격·거래대금은 1차 소스, 인증된 pykrx의 KOSPI 참여자별 순매수는 보완 소스, yfinance 조정가격은 연구용 2차 소스다. Open API 서비스 활용권한이 없거나 인증에 실패하면 인증된 pykrx KRX 가격 경로로 fail-closed 전환하되 공개 상태를 `degraded`로 두고 이유를 남긴다. 이를 Open API와 동등한 정상 상태로 표시하지 않는다. KOSPI와 수급의 최신일이 다르면 미래 행을 오래된 행에 붙이지 않고 최신 공통 거래일까지 절단하며 시차 사유를 공개한다. ETF 공식 전체 이력은 거래 세션과 다중 시점 진단에 사용하고, 최신 `dataAsOf` 종가 차이가 0.5% 이내일 때만 관련 백테스트를 공개한다. yfinance 조정가격에 공식 KRX 거래 세션이 빠진 경우 앞뒤 공통일의 `조정종가/원종가` 계수가 0.5% 이내로 일치할 때만 KRX OHLC를 같은 조정 스케일로 보정한다. 한 건이라도 보정할 수 없으면 해당 ETF 사건 연구와 백테스트 전체를 `unavailable`로 닫고, 보정·미해결 건수와 합성 출처를 공개한다. 원종가와 조정가격의 장기 차이는 배당·분할 효과가 있으므로 진단 정보이며 최신 게이트를 대체하지 않는다. 전체 백테스트는 배당·분할 반영 조정가격을 사용한다. ETF/ETN/ELW 수급은 제외한다. 미국 종가는 해당 KRX 거래일 전에 이용 가능했던 마지막 세션으로만 정렬한다.

`KRX_API_KEY`, `KRX_ID`, `KRX_PW`는 GitHub Actions Secrets 또는 일회성 환경변수로만 주입한다. 인증 구간의 stdout/stderr는 캡처하며 요청 헤더, 쿠키, 원응답과 예외 객체를 공개하지 않는다. `references/private/`, `.env*`, 세션 파일과 원시 인증 응답은 Git 및 Pages 산출물에서 제외한다.

`summary.json`은 `schemaVersion=1`, `contract=quant-research-summary`, `projectId=fearngreed`, `methodologyVersion=fear-flow-v2`을 고정한다. 엄격한 JSON Schema가 상태·범위·엔티티·robust/scaled/raw 최신 모형·포지션·대기 주문·출처·자동화·payload의 필드, 범위, enum과 추가 속성을 검증한다. 최신성은 `generatedAt`이 아니라 `dataAsOf`로 판단한다. 화면은 세 공개 JSON의 방법론 버전과 기준일이 모두 일치하지 않으면 `unavailable`로 닫히며 임의 수치나 브라우저 재계산으로 대체하지 않는다.

`dashboard.json`은 모델별 252개 학습 관측과 현재 관측 1개의 역할, 무조건부 벤치마크·초과수익을 포함한 사건 요약, ETF·비용·변형·기간별 사전 계산 백테스트, PDF 사례 격리 패널, 수급 채널 확장 상태, 공식 종가 교차검증과 반도체 상대 진단을 둔다. 각 사건 섹션의 `meanExcessReturnCi95BenchmarkTreatment`은 `fixed_external_mean`, `paired_event_returns`, `unavailable` 중 하나다. `fixed_external_mean`은 무조건부 벤치마크 평균을 고정한 조건부 bootstrap 구간으로 벤치마크 평균의 추정 불확실성을 포함하지 않는다는 뜻이다. 같은 정책을 상태·기간별 행마다 반복하지 않고 섹션 수준에 한 번 기록한다. `history.json`은 `columnar-v1` 열·행 인코딩으로 일별 파생 시계열과 세 모델의 최신 스냅샷을 공개한다. 독립 소비자가 외국인·기관의 상태 열을 거래 신호로 오해하지 않도록 `flowChannelRoles`에 개인=`primary`, 외국인·기관=`diagnostic_only`, `eligibleForTrading=false`, 활성화 규칙과 실제 열 이름을 함께 둔다. 목표 크기는 각각 500KB와 2MB 미만이며 검증기가 최소 5% 여유까지 강제한다.

갱신은 최신 5거래일을 수정 가능 구간으로 삼아 캐시를 강제 재검증하고, 수정 불가 구간의 모든 공개 열이 재계산 결과와 일치하는지 확인한다. 공개 입력을 다시 읽는 과정에서 robust z-score에 증폭되는 부동소수점 잡음만 절대오차 `5e-8`까지 허용하며 문자열·상태·포지션은 정확히 같아야 한다. Yahoo 조정가격은 경계 이전의 최대 6개 공통 앵커도 비교해 0.5%를 넘는 스케일 변경을 탐지한다. 이전 공개 이력에 검증된 KRX 보정행이 있으면 그 수와 출처를 다음 증분 실행에도 계승해, 당일 신규 보정이 0건이라는 이유로 순수 Yahoo 이력처럼 표시하지 않는다. 어느 검사든 실패하면 `requires_backfill`로 중단하며, 명시적 `backfill_start_date` 없이는 과거 원자료·신호·포지션을 일부만 고쳐 서로 다른 세대의 결과를 섞지 않는다. 검증된 결과는 임시파일을 완전히 쓰고 `fsync`한 뒤 원자적으로 교체한다. 실패하면 기존 신호·사건·백테스트를 유지하고 `summary.status`와 `automation-status.json`에 승인된 짧은 사유만 원자적으로 기록한다. 실패 실행도 이 두 파일만 검증·커밋해 공개 상태를 알린 뒤 Actions 자체는 실패로 남긴다. Pages 산출물은 `index.html`, `assets/`, `data/`, `docs/`, `schemas/`만 허용하며 `references/`는 복사하지 않는다.
